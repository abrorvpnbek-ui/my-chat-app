# 💬 Личный чат — инструкция по запуску

Мессенджер для двух человек с синхронизацией в реальном времени через Firebase.

---

## Файлы проекта

```
chat-app/
├── index.html   — структура страницы
├── style.css    — оформление
├── firebase.js  — конфиг Firebase (сюда вставляешь свои ключи)
├── app.js       — вся логика
└── README.md    — эта инструкция
```

---

## Шаг 1 — Создать проект в Firebase

1. Открой [https://console.firebase.google.com](https://console.firebase.google.com)
2. Нажми **"Создать проект"** → дай любое название → создай
3. В левом меню выбери **"Realtime Database"**
4. Нажми **"Создать базу данных"**
5. Выбери регион (например, `europe-west1`) → нажми **"Далее"**
6. Выбери **"Начать в тестовом режиме"** → **"Готово"**

> ⚠️ Тестовый режим — база открыта 30 дней. Потом нужно обновить правила (см. Шаг 3).

---

## Шаг 2 — Получить конфигурационные ключи

1. В консоли Firebase нажми на шестерёнку ⚙️ → **"Настройки проекта"**
2. Листай вниз до раздела **"Ваши приложения"**
3. Нажми иконку **`</>`** (Веб)
4. Дай приложению имя → нажми **"Зарегистрировать приложение"**
5. Скопируй объект `firebaseConfig` — он выглядит примерно так:

```js
const firebaseConfig = {
  apiKey: "AIzaSy...",
  authDomain: "my-chat-12345.firebaseapp.com",
  databaseURL: "https://my-chat-12345-default-rtdb.europe-west1.firebasedatabase.app",
  projectId: "my-chat-12345",
  storageBucket: "my-chat-12345.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123456789:web:abc123"
};
```

6. Открой файл **`firebase.js`** и вставь свои значения вместо `"ВСТАВЬ_СЮДА"`

---

## Шаг 3 — Правила безопасности (опционально, но рекомендуется)

В Firebase Console → Realtime Database → **Правила**, замени содержимое на:

```json
{
  "rules": {
    "messages": {
      ".read": true,
      ".write": true
    },
    "statuses": {
      ".read": true,
      ".write": true
    }
  }
}
```

Нажми **"Опубликовать"**.

## Шаг 4 — Разместить сайт (выбери один способ)

### Вариант А: GitHub Pages (бесплатно, 5 минут)

1. Зайди на [github.com](https://github.com) → **"New repository"**
2. Дай имя, сделай **Public**, нажми **"Create"**
3. Загрузи все 4 файла (`index.html`, `style.css`, `firebase.js`, `app.js`)
4. Перейди в **Settings** → **Pages**
5. В разделе "Branch" выбери `main` → папку `/ (root)` → нажми **"Save"**
6. Через минуту сайт будет доступен по адресу:
   `https://твой-никнейм.github.io/имя-репозитория/`

### Вариант Б: Firebase Hosting (тоже бесплатно)

```bash
npm install -g firebase-tools
firebase login
firebase init hosting
# выбери свой проект, public folder = . (точка)
firebase deploy
```

### Вариант В: Открыть локально

Просто открой `index.html` в браузере. 

> ⚠️ Важно: из-за `type="module"` в скриптах, файл **не работает** напрямую через `file://`.  
> Нужен локальный сервер. Самый простой способ:
> ```bash
> # Если установлен Python:
> python -m http.server 8080
> # Затем открой http://localhost:8080
> ```
> Или установи расширение **Live Server** в VS Code и нажми "Go Live".

## Как пользоваться

1. **Первый человек** открывает сайт → вводит имя → начинает писать
2. **Второй человек** открывает тот же URL → вводит своё имя → видит переписку
3. Сообщения появляются в реальном времени без перезагрузки

### Возможности:
- 🗺️ **Статус-локация** — кнопка в шапке, выбери где ты сейчас
- ✏️ **Смена имени** — нажми на своё имя в шапке (все старые сообщения обновятся)
- 🗑️ **Удаление** — наведи на сообщение, появится кнопка ✕
- ⏱️ **Время** — показывается под каждой группой сообщений

## Вопросы?

Если что-то не работает — скорее всего проблема с `databaseURL` в `firebase.js`.  
Убедись что он скопирован точно, включая `https://` в начале.
